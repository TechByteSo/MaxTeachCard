# MaxTeachCard

Одностраничный сайт-визитка педагога (статический HTML/CSS/JS).

## Локальный запуск

Откройте `index.html` в браузере или поднимите локальный сервер (например, Live Server в VS Code / Cursor), чтобы корректно работали некоторые сценарии.

## Деплой

- Залейте содержимое репозитория на **GitHub Pages**, **REG.RU** или любой статический хостинг с **HTTPS**.
- После смены базового URL обновите **одинаковый** адрес во всех местах:
  - `index.html`: `og:url`, `og:image`, `canonical`, блок `<script type="application/ld+json">`, при необходимости `twitter:image`;
  - `sitemap.xml` — элемент `<loc>`;
  - `robots.txt` — строка `Sitemap:`.
- На Linux-серверах учитывайте **регистр** имён файлов и папок (в т.ч. кириллица в `images/` и в ссылках на PDF в `documents/` — имена файлов должны совпадать с реальными на сервере).
- В проекте используйте единые каталоги изображений для контента портфолио:
  - `images/Городские уличные фестивали Москвы 2025/`
  - `images/День города Москва 875/`
  - `images/Участие в БГУ/`
  - `images/Стажировка Волгоград/`

## Портфолио (новый проект в блоке «Показать все»)

1. Скопируйте целый блок `events__card` с соседней карточки внутри `#eventsMoreWrapper` / `.events__more-inner` (перед закрывающим `</div>` этой внутренней обёртки).
2. Задайте у трека **следующий по порядку** `id="sliderN"` и тот же идентификатор в `data-slider` у стрелок и точек; индексы точек — `0 … N-1` по числу слайдов (см. текущую нумерацию в `index.html`).
3. Обновите пути к изображениям, `alt`, заголовки и подписи в overlay. У разных карточек не должно быть двух треков с одним `id`.

## Контакты на сайте

Телефон (`tel:`), ссылка на **Telegram**; запрос e-mail — через ту же переписку в Telegram (отдельного рабочего `mailto:` в разметке нет, пока не добавите свой).

## Документы (PDF)

Файлы из вкладки «Документы» ожидаются в каталоге `documents/` с теми же именами, что в `href` (кириллица и пробелы учитываются). При отсутствии файлов ссылки на хостинге дадут 404.

## Качество и доверие (перед релизом)

- **Lighthouse** (Chrome DevTools → вкладка Lighthouse): доступность, SEO, производительность, best practices.
- **PageSpeed Insights**: [pagespeed.web.dev](https://pagespeed.web.dev/) — внешняя проверка и рекомендации по LCP/CLS.
- **Разметка HTML**: [W3C Markup Validator](https://validator.w3.org/nu/) (для локального файла — загрузка или проверка после публикации по URL).
- **Консоль и сеть**: нет красных ошибок в Console; в Network нет неожиданных 404 на CSS, JS, шрифты, картинки, PDF.
- Полный чеклист: [DoD for Landing.md](./DoD%20for%20Landing.md).

На странице уже подключены **структурированные данные** Schema.org (`Person` + `WebSite` в JSON-LD), Open Graph, **Twitter Card** (`summary_large_image`), `sitemap.xml` и ссылка на него в `robots.txt` (URL по умолчанию под GitHub Pages; при своём домене замените).

## Техника

### Минификация

```bash
npm install
npm run minify
```

Команда пересобирает:

- `styles/style.min.css`
- `styles/slider.min.css`
- `scripts/script.min.js`

После правок в `style.css`, `slider.css` или `script.js` **всегда** коммитьте и обновлённые `*.min.*`, иначе на сайте останется старая версия.

### CI (GitHub Actions)

Workflow [.github/workflows/verify-minified.yml](.github/workflows/verify-minified.yml) на `push`/`pull_request` в `main`/`master` выполняет `npm ci`, `npm run minify` и падает, если закоммиченные минифицированные файлы **не совпадают** с выводом сборки. Так проще не забыть пересобрать ассеты перед merge.

### Шрифты и иконки (CDN)

В `<head>` добавлены `dns-prefetch` / `preconnect` для Google Fonts и cdnjs (ускорение первого запроса). Полный self-host шрифтов и Font Awesome возможен, но увеличивает размер репозитория — делайте осознанно при жёстких требованиях к автономности.

### Фавиконы

Сейчас в репозитории есть `favicon.svg`. При появлении `favicon.ico`, PNG и `apple-touch-icon.png` можно снова добавить соответствующие `<link>` в `<head>`.

## Продукт: что держать в голове при правках

| Зона | Что проверить |
|------|----------------|
| Портфолио | Новые webp в `images/…`, уникальный `sliderN`, стрелки/точки с тем же `data-slider`, клик по фото → лайтбокс |
| Документы | Файл в `documents/` + карточка с тем же `href`; открытие в новой вкладке уже с `rel="noopener noreferrer"` |
| Домен | Синхронно: `canonical`, OG, JSON-LD, `sitemap.xml`, `robots.txt` |
| Доступность | Подписи у кнопок-иконок, фокус с клавиатуры, контраст (см. DoD) |
